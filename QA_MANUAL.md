# QA Manual — WhatsApp Bot NLU Parity (Phases 1-6)

**Session:** 2026-07-05
**Scope:** All NLU / user-intent changes shipped in Phases 1-6 (D5-T9 → D5-T13)
**Environment:** Production WhatsApp (Galit CRM bot)
**Estimated time:** ~45-60 min end-to-end

---

## הוראות כלליות

- כל בדיקה מתועדת כ־Test Case (TC) עם צעדים, תוצאה מצופה, ותוצאה בפועל שתמלא.
- אחרי כל בדיקה, סמן ✅ / ❌ / ⚠️ ורשום הערות אם משהו לא ברור.
- אם צריך להחליף role/name כדי לבדוק פלואו של עובד — שנה ב־User table ולחץ "start over" ב־WhatsApp אם צריך.
- מומלץ להתחיל בסביבת פרודקשן כי לא נעשה smoke live.

**מדינות של המשתמש הבודק:**
- **Worker mode:** role=TECHNICIAN, name=אופק (או כל שם עברי אחר שאינו יורם/סשה/גיא פרנסס/גיא גבאי/יאיר)
- **Manager mode:** role=ADMIN או MANAGER, name=גיא פרנסס (או כל שם מנהל)
- **Sasha mode (לידים):** name=סשה

### TC-2.6 — Free-text בכל מצב detail-view (D5-T15 hotfix)
**Setup:** מצב עובד (או מנהל שהוא גם עובד). היכנס ל־detail view של בדיקה ספציפית:
1. שלח `הבדיקות שלי` → רשימת בדיקות ממוספרת
2. הקש `1` (או כל מספר בטווח) → detail view של הבדיקה + תפריט 4 פעולות:
   1. תיקון פרטי ביקור  2. תיקון סוג בדיקה  3. שיוך מחדש  4. חזרה

**במצב הזה** (state = `mgr_today_action`), בדוק כל אחת מהפראזות. **לפני התיקון:** הבוט היה מחזיר "לא זוהתה פעולה ברורה מההודעה". **אחרי התיקון:** הבוט מבצע את הפעולה על הבדיקה שאתה רואה.

- `יצאתי` → סטטוס עולה ל־EN_ROUTE
- `בדרך` / `בדרכי` → EN_ROUTE
- `הגעתי` / `אני באתר` → ARRIVED
- `סיימתי` / `גמרתי` → FINISHED_FIELD + תפריט המשך של 4 (finished_followup)
- `הלקוח לא ענה` → כתיבת HAS_PROBLEM עם problem_type=CUSTOMER_NOT_ANSWERING + התראה למשרד
- `אין תשובה` → אותו דבר
- `אין גישה` → HAS_PROBLEM / NO_ACCESS
- `יש לי בעיה` (בלי סוג) → פותח sub-menu של 7 סוגי הבעיות עבור הבדיקה הנוכחית
- `שכחתי את המדד` → WAITING_FOR_INFO / כתיבת note "המדד" + התראה
- `חסר לי טופס דגימה` → WAITING_FOR_INFO / כתיבת note "טופס דגימה"

**מצופה:**
- **לא** מקבל "לא זוהתה פעולה ברורה"
- הפעולה מתבצעת על ה־TaskField שאתה רואה — לא נשאל "לאיזה בדיקה?"
- אישור התאמה לפעולה

**גם — regression:** בדוק ש־תיקונים עדיין עובדים ב־detail view:
- `שנה את הכתובת לרוטשילד 20 תל אביב` → correct_site flow (לא set_field_status)
- `שייך מחדש לדני` → reassign flow
- `שנה סוג בדיקה לבדיקת קרינה` → correct_type flow

**תוצאה בפועל:** _______________________________________________

**Reset context (WhatsApp side):** אין פקודה ישירה, אבל שליחת `היי` תמיד פותחת תפריט נקי.

**Change user role (DB side):**
```sql
-- Set as worker
UPDATE "User" SET role = 'TECHNICIAN', name = 'אופק' WHERE id = '<your-user-id>';

-- Set as manager
UPDATE "User" SET role = 'ADMIN', name = 'גיא פרנסס' WHERE id = '<your-user-id>';

-- Set as Sasha (leads-viewer)
UPDATE "User" SET name = 'סשה' WHERE id = '<your-user-id>';
```

**Reset a TaskField to open status (for repeat testing):**
```sql
UPDATE "TaskField" SET
  "fieldStatus" = 'ASSIGNED',
  "problemType" = NULL,
  "problemNote" = NULL,
  "missingReportInfoNote" = NULL,
  "confirmedAt" = NULL,
  "arrivedAt" = NULL,
  "finishedAt" = NULL
WHERE id = '<taskFieldId>';
```

**Create 3 open TaskFields for the same worker (for disambig testing):**
